# 数据驱动项目系统用户操作手册

## 一、系统概述

数据驱动项目系统是一个基于数据感知和驱动逻辑的智能系统，支持通过配置实现数据的实时监控和自动化处理。系统提供了直观的用户界面，方便用户进行数据源管理、业务模型配置、数据感知规则设置、驱动逻辑定义和Agent管理等操作。

## 二、系统访问

### 1. 后端服务
- **地址**: http://127.0.0.1:8080
- **API文档**: http://127.0.0.1:8080/docs

### 2. 前端应用
- **地址**: http://localhost:3000/

## 三、功能模块使用指南

### 1. 数据源管理

#### 1.1 添加数据源
1. 点击左侧菜单中的"数据源管理"
2. 点击"添加数据源"按钮
3. 填写数据源信息：
   - 名称：数据源的名称
   - 类型：选择SQLite
   - 连接字符串：SQLite数据库文件路径，格式为 `sqlite:///path/to/database.db`
   - 描述：可选，数据源的描述信息
4. 点击"确定"按钮
5. 点击"测试连接"按钮验证连接是否成功

#### 1.2 管理数据源
- **编辑**: 点击数据源行的"编辑"按钮修改信息
- **删除**: 点击数据源行的"删除"按钮删除数据源
- **查看表**: 点击"测试连接"按钮后，可以获取数据源中的表列表

### 2. 业务模型管理

#### 2.1 导入模型
1. 点击左侧菜单中的"业务模型管理"
2. 点击"导入模型"按钮
3. 选择数据源
4. 选择要导入的表
5. 可选：填写模型ID（不填写则使用表名）
6. 点击"确定"按钮
7. 系统会自动识别表结构并生成业务模型，包括：
   - 模型ID（表名）
   - 中文名称（通过LLM生成）
   - 中文说明（通过LLM生成）
   - 主键ID（自动识别）
   - 字段信息（自动识别并生成中文名称和说明）

#### 2.2 手动创建/编辑模型
1. 点击"添加模型"按钮
2. 填写模型信息：
   - 模型ID：唯一标识符
   - 中文名称：模型的中文名称
   - 中文说明：模型的描述信息
   - 主键ID：模型的主键字段
   - 数据源：关联的数据源
3. 点击"确定"按钮

### 3. 数据感知配置

#### 3.1 添加数据变化感知
1. 点击左侧菜单中的"数据感知配置"
2. 点击"添加配置"按钮
3. 填写配置信息：
   - 名称：配置的名称
   - 类型：选择"数据变化感知"
   - 业务模型：选择要监控的业务模型
   - 配置参数：JSON格式，例如：
     ```json
     {
       "trigger_conditions": ["create", "update", "delete"]
     }
     ```
   - 描述：配置的描述信息
4. 点击"确定"按钮

#### 3.2 添加阈值触发感知
1. 点击"添加配置"按钮
2. 填写配置信息：
   - 名称：配置的名称
   - 类型：选择"阈值触发感知"
   - 业务模型：选择要监控的业务模型
   - 配置参数：JSON格式，例如：
     ```json
     {
       "field": "temperature",
       "operator": ">",
       "threshold": 100,
       "duration": 30
     }
     ```
   - 描述：配置的描述信息
3. 点击"确定"按钮

### 4. 驱动逻辑配置

#### 4.1 添加一阶函数
1. 点击左侧菜单中的"驱动逻辑配置"
2. 点击"添加驱动逻辑"按钮
3. 填写逻辑信息：
   - 名称：逻辑的名称
   - 类型：选择"一阶函数"
   - 配置参数：JSON格式，例如：
     ```json
     {
       "expression": "data.value > 100"
     }
     ```
   - 描述：逻辑的描述信息
4. 点击"确定"按钮

#### 4.2 添加脚本函数
1. 点击"添加驱动逻辑"按钮
2. 填写逻辑信息：
   - 名称：逻辑的名称
   - 类型：选择"脚本函数"
   - 配置参数：JSON格式，例如：
     ```json
     {
       "script": "def check(data): return data['value'] > 100 and data['status'] == 'active'"
     }
     ```
   - 描述：逻辑的描述信息
3. 点击"确定"按钮

#### 4.3 添加任务
1. 点击"添加任务"按钮
2. 填写任务信息：
   - 名称：任务的名称
   - 类型：任务的类型
   - 配置参数：JSON格式，例如：
     ```json
     {
       "target": "email",
       "recipient": "user@example.com",
       "subject": "警报通知"
     }
     ```
   - 描述：任务的描述信息
3. 点击"确定"按钮

### 5. Agent管理

#### 5.1 添加Agent
1. 点击左侧菜单中的"Agent管理"
2. 点击"添加Agent"按钮
3. 填写Agent信息：
   - 名称：Agent的名称
   - 描述：Agent的描述信息
   - 状态：选择"活跃"或"非活跃"
4. 点击"确定"按钮

#### 5.2 添加能力
1. 点击"添加能力"按钮
2. 填写能力信息：
   - 名称：能力的名称
   - 任务类型：能力对应的任务类型
   - 描述：能力的描述信息
3. 点击"确定"按钮

#### 5.3 分配能力给Agent
1. 在Agent列表中，点击"分配能力"按钮
2. 选择要分配的能力
3. 点击"确定"按钮

### 6. 测试数据管理

#### 6.1 查看测试数据
1. 点击左侧菜单中的"测试数据管理"
2. 选择数据源
3. 选择表名
4. 系统会显示表中的数据

#### 6.2 添加测试数据
1. 点击"添加数据"按钮
2. 填写数据字段
3. 点击"确定"按钮

### 7. 驱动日志

#### 7.1 查看日志
1. 点击左侧菜单中的"驱动日志"
2. 系统会显示所有驱动日志

#### 7.2 筛选日志
- **级别**: 选择日志级别（Info、Warning、Error）
- **分类**: 选择日志分类（数据感知、驱动逻辑、Agent任务）
- **搜索**: 输入关键词搜索日志消息

### 8. 测试执行

#### 8.1 模拟事件
1. 点击左侧菜单中的"测试执行"
2. 选择数据感知配置
3. 填写事件数据（JSON格式）
4. 点击"模拟事件"按钮
5. 查看执行结果

## 四、系统操作流程示例

### 示例1：配置数据变化感知

2. **导入业务模型**: 选择表并导入生成业务模型

4. **添加驱动逻辑**: 创建驱动逻辑，设置处理规则
5. **添加任务**: 创建要执行的任务

7. **模拟事件**: 测试阈值触发和处理流程

## 五、常见问题

### 1. 数据源连接失败
- 检查连接字符串是否正确
- 确保SQLite数据库文件存在且可访问
- 检查数据库文件权限

### 2. 业务模型导入失败
- 确保数据源连接正常
- 确保表存在于数据源中
- 检查数据库权限

### 3. 事件不触发
- 检查数据感知配置是否正确
- 确保数据满足触发条件
- 检查数据感知引擎是否运行

### 4. 驱动逻辑不执行
- 检查驱动逻辑配置是否正确
- 确保事件与驱动逻辑正确关联
- 检查驱动引擎是否运行

### 5. Agent任务不执行
- 检查Agent状态是否为活跃
- 确保Agent具有执行任务的能力
- 检查任务配置是否正确

## 六、系统维护

### 1. 数据库备份
- 定期备份SQLite数据库文件（data.db）

### 2. 日志管理
- 定期清理驱动日志，避免日志文件过大

### 3. 系统重启
- 当系统出现异常时，可以重启后端服务
- 重启命令：`python -m app.main`
