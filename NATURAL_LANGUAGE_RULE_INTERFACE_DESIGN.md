# 自然语言规则接口功能设计文档

## 1. 功能概述

自然语言规则接口功能允许用户使用自然语言描述业务规则需求，系统自动将其转换为结构化的数据感知配置和驱动逻辑配置。同时，系统也能将现有的结构化配置以自然语言形式呈现，提高配置的可读性和易用性。

### 核心价值
- **降低技术门槛**: 业务人员无需了解技术细节，可以用自然语言描述需求
- **提高配置效率**: 自动化生成配置，减少手动配置工作量
- **增强可读性**: 将技术配置转换为业务友好的自然语言描述
- **双向交互**: 支持自然语言 ↔ 结构化配置的双向转换

## 2. 系统架构

### 2.1 整体流程
```
自然语言输入 → LLM理解 → 结构化配置 → 验证 → 应用配置
结构化配置 → LLM解释 → 自然语言描述 → 用户展示
```

### 2.2 组件关系
- **前端**: 在现有配置页面集成自然语言输入框和解析按钮
- **后端API**: nl_rule_interface.py（四个核心接口）
- **服务层**: 扩展LLMTranslator类，添加自然语言处理方法
- **数据层**: 复用现有的数据感知配置和驱动逻辑数据模型

## 3. 详细设计

### 3.1 自然语言解析模式

#### 3.1.1 数据感知配置支持的自然语言模式

**数据变化感知**:
- "监控订单表的所有变更" 
- "当客户信息更新时触发"
- "监听产品库存的新增和删除操作"

**阈值触发感知**:
- "当温度超过100度时告警"
- "当库存低于50时通知"
- "当订单金额大于10000时触发审批流程"

#### 3.1.2 驱动逻辑配置支持的自然语言模式

**一阶函数**:
- "如果订单金额大于10000，则需要经理审批"
- "当温度异常时发送邮件通知"
- "如果客户等级为VIP，则优先处理"

**脚本函数**:
- "计算风险评分并根据结果分配不同处理流程"
- "聚合多个数据源的信息并生成综合报告"

### 3.2 LLM提示词设计

#### 3.2.1 解析提示词设计原则
- **明确角色定义**: "专业的数据驱动系统配置专家"
- **提供上下文信息**: 可用的业务模型、字段、任务等
- **指定输出格式**: 严格的JSON schema，便于解析验证
- **包含错误处理**: 无法解析时返回空对象或错误信息

#### 3.2.2 解释提示词设计原则
- **简洁明了**: 使用业务语言，避免技术术语
- **一致性**: 相同配置生成相似的描述
- **完整性**: 包含所有关键配置信息

### 3.3 后端API接口 (app/api/nl_rule_interface.py)

**四个核心接口**:

1. **POST /nl-rule-interface/parse-sensing-config**
   - 功能: 将自然语言解析为数据感知配置
   - 参数: natural_language (string)
   - 返回: {success: bool, config: dict}

2. **POST /nl-rule-interface/parse-drive-logic**
   - 功能: 将自然语言解析为驱动逻辑配置  
   - 参数: natural_language (string)
   - 返回: {success: bool, logic: dict}

3. **POST /nl-rule-interface/explain-sensing-config**
   - 功能: 将数据感知配置转换为自然语言描述
   - 参数: config (dict)
   - 返回: {success: bool, explanation: string}

4. **POST /nl-rule-interface/explain-drive-logic**
   - 功能: 将驱动逻辑配置转换为自然语言描述
   - 参数: logic (dict)
   - 返回: {success: bool, explanation: string}

## 4. 扩展LLMTranslator类

### 4.1 新增方法

**解析方法**:
- `parse_natural_language_to_sensing_config()`: 自然语言 → 数据感知配置
- `parse_natural_language_to_drive_logic()`: 自然语言 → 驱动逻辑配置

**解释方法**:
- `convert_sensing_config_to_natural_language()`: 数据感知配置 → 自然语言
- `convert_drive_logic_to_natural_language()`: 驱动逻辑配置 → 自然语言

### 4.2 提示词构建方法

- `_build_sensing_config_parsing_prompt()`: 构建数据感知配置解析提示词
- `_build_drive_logic_parsing_prompt()`: 构建驱动逻辑解析提示词  
- `_build_sensing_config_explanation_prompt()`: 构建数据感知配置解释提示词
- `_build_drive_logic_explanation_prompt()`: 构建驱动逻辑解释提示词

## 6. 错误处理与容错

### 6.1 解析错误处理
- **模糊输入**: 提供可能的解析选项供用户选择
- **无法识别**: 返回具体的错误信息和示例
- **格式错误**: 捕获JSON解析异常，提供友好的错误提示

### 6.2 性能优化
- **缓存机制**: 缓存常用的自然语言解析结果
- **异步处理**: 对于复杂的自然语言输入，使用异步处理
- **输入限制**: 限制自然语言输入长度，防止过长输入

### 6.3 安全考虑
- **输入验证**: 防止恶意自然语言输入
- **输出验证**: 验证LLM返回的配置是否符合安全要求
- **访问控制**: 确保只有授权用户可以使用此功能
