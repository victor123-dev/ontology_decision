# 半导体仿真系统架构说明

## 系统概述

本系统是一个基于SimPy离散事件仿真的半导体制造业APS+MRP仿真引擎，模拟通富微电等OSAT封装测试工厂的真实生产场景。

**核心特性**：
- 事件驱动实时排程机制
- Lot批量加工（25个芯片/批）
- 良率损耗传递
- 100-135道工序/产品的复杂工艺路线
- 120天完整生产周期仿真

---

## 一、排程机制

### 1.1 双模式排程架构

系统采用**实时事件驱动 + 定时全局扫描**的双模式排程机制：

#### 模式一：实时事件驱动排程
- **触发时机**：每道工序完成时立即触发
- **方法**：`realtime_schedule_after_completion()`
- **作用**：立即排程因前驱完成而变为ready的后续工序
- **响应速度**：<1分钟
- **智能过滤**：只排程相关工序，避免全局扫描开销

#### 模式二：定时全局排程
- **触发时间**：每天09:00 + 每4小时检查一次（09:00, 13:00, 17:00, 21:00）
- **方法**：`daily_scheduler()`
- **作用**：全局扫描所有可排工序，处理积压任务
- **工作日历**：跳过周日不排程

### 1.2 排程算法流程

```
1. 查询可排工序（前驱已完成 + 物料齐套）
2. 按关键比排序（Critical Ratio = 剩余时间/剩余工期）
3. 按工作中心分组（work_center_id）
4. 对每个工作中心：
   - 遍历候选机台
   - 计算完工时间 + 换线时间
   - 选择最优机台（得分最低）
5. 创建production_task
6. 启动execute_production_task事件
```

### 1.3 关键比排序算法

```python
critical_ratio = 剩余时间 / (剩余工序数 × 4.0)
优先级排序：(critical_ratio, priority) 升序
```

- 关键比越小越紧急
- 关键比 < 1.0 表示工期紧张
- 同关键比时按订单优先级排序（1最紧急）

## 二、工序执行流程

### 2.1 工序计划时间计算

**Lot批量加工机制**：
- 每道工序的标准时间已按Lot批量（25个芯片/批）调整
- 计划时间直接使用调整后的`std_time`
- 不重复乘以批量系数

**时间偏移累积**：
```python
cumulative_offset = 4.0小时  # 起始偏移
for 每道工序:
    planned_start = now + cumulative_offset
    planned_duration = std_time  # 批量调整后的时间
    planned_end = planned_start + std_time + wait + transport
    
    # 累积到下一道工序
    queue_time = 0.5小时  # 排队时间
    cumulative_offset += std_time + wait + transport + queue_time
```

### 2.2 生产执行步骤

```
1. 等待到planned_start_time
2. 等待transport_time（转运时间）
3. 机台Setup（如有换线）
4. IPQC首件检验（换线后）
5. 记录实际开始时间
6. 生产领料（预留→消耗）
7. 加工（timeout process_time）
8. 计算良率损耗
9. 记录实际结束时间
10. 更新工序状态="已完成"
11. 触发实时排程（排程后续工序）
12. 推进WIP Lot到下一工序
13. 更新工单状态
```

### 2.3 良率损耗传递

每道工序完工后：
```python
actual_yield = 机台良率 × 工序标准良率 × 随机扰动(0.99~1.01)
actual_qty = input_qty × actual_yield
下一道工序投入量 = actual_qty
```

## 三、工单生命周期

### 3.1 订单生成

```python
每天订单数 = Poisson分布(lambda=3.0)
订单数量 = 随机5-80个芯片
优先级 = 随机(1:20%, 3:30%, 5:50%)
交期承诺 = max(CTP估算, 订单日期+7天)
```

### 3.2 工单创建

创建工单后执行三步：
1. **展开工序**（expand_work_order_operations）
2. **展开物料需求**（expand_work_order_materials）
3. **创建WIP Lot**（create_wip_lots）

### 3.3 WIP Lot管理

```
Lot大小 = 25个芯片/批
订单50个芯片 → 2个Lot（25+25）
订单71个芯片 → 3个Lot（25+25+21）
```

- 一个工单有多个Lot
- 每个Lot独立走完所有工序
- 工序记录共用一份（work_order_operation）
- 通过production_task区分不同Lot的执行

### 3.4 工单状态流转

```
已下达 → 生产中 → 已完成
               ↓
              延期（超计划完工日期）
```

**状态更新逻辑**：
```python
if completed_ops == 0:
    status = "已下达"
elif completed_ops < total_ops:
    status = "生产中"
else:
    status = "已完成"
    调用完工入库和发货流程
```

## 四、物料管理

### 4.1 MRP运算

**触发时间**：每天08:30

**逻辑流程**：
```
1. 查询所有"待分配"物料需求
2. 按工单优先级排序
3. 对每条需求：
   - 有库存 → 直接预留
   - 库存不足 → 找替代料
   - 仍不足 → 从其他工单调拨
   - 最后手段 → 触发采购
```

### 4.2 物料状态

- **待分配**：尚未分配库存
- **已齐套**：已预留足够库存
- **部分分配**：部分预留，有缺口
- **缺料**：无法分配
- **已消耗**：工序开工时从预留转为消耗

### 4.3 工序开工领料

触发时机：`execute_production_task`开始时
```python
查询该工序的物料需求（wo_op_id匹配）
for 每个已预留的物料:
    allocated → consumed
```

## 五、质检机制

### 5.1 IPQC首件检验

- **触发**：每次换线Setup完成后
- **不合格率**：15%
- **处理**：不合格时调整0.5小时后重检
- **记录**：`quality_inspection`（type="首件检验"）

### 5.2 FQC成品检验

- **触发**：每次发货前
- **不合格率**：5%
- **处理**：不合格时批次Hold、退回库存
- **记录**：`quality_inspection`（type="FQC出货检验"）

### 5.3 关键工序质检

- 关键工序完成后触发过程质检
- 可能改变最终报废数量

## 六、机台管理

### 6.1 机台匹配规则

**排程使用`work_center_id`匹配机台**：
```python
候选机台 = 工作中心ID匹配 AND 机台未维护
```

**注意**：`machine_type`字段是预留字段（如"自动"），当前未用于排程逻辑。

### 6.2 机台故障与维护

**随机故障**：
- 70%机台启用随机故障
- MTBF=96小时（指数分布）
- MTTR=4小时（均匀分布1~8小时）

**计划维护（PM）**：
- 每168小时（7天）一次
- 维护时间：2~6小时

### 6.3 换线机制

**换线矩阵**（setup_matrix）：
- A产品→B产品：换线时间可能不同
- 同SetupGroup：换线时间=0
- 换线后触发IPQC首件检验

### 6.4 OEE动态更新

**更新时机**：每天20:30
**计算**：
- 效率 = 实际加工时间 / 标准时间
- 良率 = 合格品数量 / 总加工数量
- 更新到`machine_capability`表

## 七、工作日历

### 7.1 工作时间

- **日班**：08:00-20:00
- **夜班**：20:00-次日08:00（效率折减0.92）
- **周日**：完全不排程

### 7.2 周日处理

```python
if today.weekday() == 6:  # 周日
    跳过排程
```

- `skip_sunday()`方法确保任务不落在周日
- 自动推迟到周一08:00

## 十二、优化建议

### 12.1 WC-TEST-BI机台瓶颈

**瓶颈分析**：
- BI工序时间：0.96小时/道（其他工序0.081小时，12倍差距）
- 当前配置：4台，理论利用率142%
- 实际影响：实时排程缓解，订单仍能按时完成

**优化方案**（可选）：
```python
MACHINE_CFG["WC-TEST-BI"] = 6  # 增加到6台，利用率降至94.7%
```

### 12.2 排队时间调整

当前排队时间：0.5小时
- 适用于Lot批量加工场景
- 如需进一步压缩周期，可减少到0.3小时
- 但需考虑实际生产约束
