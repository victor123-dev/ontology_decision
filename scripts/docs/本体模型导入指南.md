## ✅ 特性

### 1. 默认清空模式
- **默认行为**：先清空所有数据，再全量导入
- **清空顺序**：关系 → 字段 → 模型（依赖关系倒序）
- **适用场景**：确保每次都是干净的状态，避免数据不一致

### 2. 幂等模式（可选）
- 使用 `--no-clear` 参数切换
- 跳过已存在的记录，不删除现有数据
- **适用场景**：增量更新、部分导入

### 3. 全量维护
- 所有关系集中在一个文件中
- 不再需要差量修复脚本

### 4. 业务关系完整
- 不仅包含数据库外键关系
- 包含中间表的1:N关系（如BOM、能力矩阵等）
- 包含所有业务语义关系

## 📝 关系设计原则

### 不仅限于外键

本体关系 ≠ 数据库外键。关系设计基于**业务语义**：

**示例1：BOM中间表**
```
数据库外键：
  - BOM.product_id → Product.product_id
  - BOM.material_id → Material.material_id
  - BOM.step_id → RouteStep.step_id

本体关系：
  - R3_has_bom: Product ↔ Material (N:N，通过BOM)
  - R3_1_product_has_bom: Product → BOM (1:N)
  - R3_2_bom_has_material: BOM → Material (N:1)
  - R3_3_step_has_bom: RouteStep → BOM (1:N)
```

**示例2：能力矩阵中间表**
```
数据库外键：
  - MachineCapability.machine_id → Machine.machine_id
  - MachineCapability.product_id → Product.product_id

本体关系：
  - R5_capable_of: Machine ↔ Product (N:N，通过能力矩阵)
  - R5_1_machine_has_capability: Machine → MachineCapability (1:N)
  - R5_2_product_has_capability: Product → MachineCapability (1:N)
```
