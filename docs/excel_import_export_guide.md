# Excel导入导出工具使用指南

## 概述

Excel导入导出工具提供了一套完整的解决方案，用于在业务模型层与业务实例层之间进行双向数据迁移。该工具支持将系统中的业务模型配置、关系配置、行动配置以及实例数据导出到标准Excel文件，并支持从符合规范的Excel文件中导入数据回系统。

## 功能特性

### 1. 导出至Excel功能

- **多页签结构**：生成包含多个页签的标准Excel文件，页签顺序严格遵循：对象页签→关系页签→行动页签→各对象实例数据页签
- **完整配置信息**：
  - 对象页签：导出SQLite数据库中`business_models`表和`business_model_fields`表的配置信息
  - 关系页签：导出SQLite数据库中`business_model_links`表的配置信息  
  - 行动页签：导出MongoDB中`actions`集合的所有行动相关数据
  - 实例数据页签：每个对象对应一个独立页签，页签名称与对象中文名称保持一致
- **双行表头设计**：实例数据页签采用双行表头，首行为英文字段名，次行为中文说明

### 2. 从Excel导入功能

- **格式验证**：支持读取符合导出规范的Excel文件，确保数据格式正确
- **数据校验**：实现完整的数据校验机制，确保导入数据的完整性、一致性和有效性
- **按序导入**：按照页签顺序依次导入数据至对应的存储位置
- **结果反馈**：提供详细的导入结果反馈，包括成功导入记录数、失败记录数及失败原因详情

### 3. 异常处理与性能优化

- **异常处理**：处理文件格式错误、数据冲突、权限问题等异常情况，并提供友好提示
- **大数据量支持**：支持大数据量的导入导出处理，采用流式处理和内存优化策略
- **可维护性**：代码结构清晰，具备良好的可维护性和可扩展性

## API接口

### 导出接口

**POST** `/api/v1/excel/export`

**功能**：将业务模型和实例数据导出到`docs/business_model_export.xlsx`文件

**响应**：JSON格式的成功消息，包含文件路径

**示例**：
```bash
curl -X POST "http://localhost:8080/api/v1/excel/export"
```

### 导入接口

**POST** `/api/v1/excel/import`

**功能**：从`docs/business_model_import.xlsx`文件导入业务模型和实例数据

**响应**：JSON格式的导入结果，包含详细的成功/失败统计

**示例**：
```bash
curl -X POST "http://localhost:8080/api/v1/excel/import"
```

## 使用注意事项

### 1. 文件路径和命名

- **导出文件**：自动生成到`docs/business_model_export.xlsx`
- **导入文件**：需要手动准备`docs/business_model_import.xlsx`文件
- 系统会自动创建`docs`目录（如果不存在）      

### 2. 文件格式要求

- 仅支持`.xlsx`格式的Excel文件
- 页签名称必须严格按照规范：`对象`、`关系`、`行动`，以及各对象的中文名称
- 实例数据页签必须包含双行表头（英文字段名 + 中文说明）

### 3. 数据一致性

- 导入时会自动处理数据冲突（存在相同ID的记录会被更新）
- 确保业务模型、关系、行动的引用完整性
- 实例数据导入前需要确保对应的业务模型已存在

### 4. 操作流程

**导出流程**：
1. 调用导出API
2. 系统在`docs`目录下生成`business_model_export.xlsx`文件
3. 用户可以直接使用该文件

**导入流程**：
1. 用户准备`business_model_import.xlsx`文件并放置在`docs`目录下
2. 调用导入API
3. 系统读取文件并执行导入操作
4. 返回详细的导入结果

### 3. 性能考虑

- 对于大数据量导出，建议在非高峰时段执行
- 导入大文件时可能需要较长时间，请耐心等待
- 系统会自动清理临时文件，无需手动干预

### 4. 错误处理

- 如果导入失败，系统会保留原有数据不变
- 详细的错误信息会在响应中返回，便于排查问题
- 建议先在测试环境中验证Excel文件的正确性

## 扩展性说明

该工具设计具有良好的扩展性：

1. **新增业务模型类型**：只需在相应的数据表中添加配置，导出功能会自动包含
2. **调整数据结构**：Excel模板会根据实际的数据库结构动态生成
3. **自定义验证规则**：可以在服务层添加额外的数据验证逻辑
4. **支持其他数据源**：通过扩展`DataSourceManager`可以支持更多类型的数据库
